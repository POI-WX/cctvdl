# 常见问题 / FAQ

操作步骤见 [使用指南](USAGE.md)；这里收集安装与使用中的常见问题与排查。

## 🚀 安装与启动

**1. macOS 提示「无法验证开发者」或「已损坏，无法打开」？**
安装包未做 Apple 开发者签名（仅 ad-hoc 自签名），首次打开会被 Gatekeeper 拦截。两种放行方式任选其一：

- **系统设置（推荐）**：打开「系统设置 → 隐私与安全性」，在底部点击「仍要打开」。
- **终端命令**：执行以下命令清除「网络下载」隔离标记后再打开（若系统设置里没有「仍要打开」按钮，多见于旧版安装包，用这种方式）：

  ```bash
  xattr -dr com.apple.quarantine "/Applications/cctvdl.app"
  ```

> 备注：命令里的 `cctvdl.app` 是装进「应用程序」后的程序名，与版本号无关（带版本号的是下载的 `.dmg` 安装包）。终端命令仅清除 Gatekeeper 的下载隔离标记以绕过首次检查，请务必确认是从本项目官方 [Releases](../../releases) 下载后再执行；对来路不明的安装包不要这样做。

**2. Windows SmartScreen 拦截安装程序？**
同样因为安装包未签名。点击「更多信息 → 仍要运行」即可。

**3. 需要自己安装 Node 或 ffmpeg 吗？**
不需要。安装包已内置运行时与 ffmpeg，开箱即用。仅**从源码构建**时才需要 Node 22.x（见 [贡献指南](../CONTRIBUTING.md)），ffmpeg 也会随 `npm install` 自动获取。

## 🔍 导入与浏览

**4. 粘贴链接后提示「无法解析节目信息」？**
请直接复制浏览器地址栏中的央视节目页或栏目页链接（`tv.cctv.com` 下的页面），不要手动拼接。个别改版页面可能无法抽取栏目信息，可换用该栏目的标准首页链接。

**5. 视频列表为空？**
该栏目当月可能没有更新，切换月份再试；部分内容以「专辑」组织，程序会自动回退到专辑接口。

## ⬇️ 下载

**6. 下载的文件保存在哪里？是什么格式？**
保存到「设置 → 文件保存位置」指定的目录，格式为 **MP4**。完成后可在下载页一键打开所在文件夹。

**7. 下载完怎么确认视频没问题？**
在「已完成」卡片点「▶ 播放」，用系统默认播放器打开；能正常播放即说明下载与解密都成功了。

**8. 最高能下到多清晰？支持 4K 吗？**
可下载的最高档为 **1080p 蓝光**——即使是 4K 频道的内容，也按 1080p 蓝光下载。清晰度选「蓝光」或「自动」即可取到该节目提供的最高档；不同节目实际提供的档位不同，缺档会自动降级。

**9. 提示「未设置保存位置」/「保存目录不可写」？**
在「设置」中选择一个有写权限的目录（避免系统盘受保护目录）。

**10. 提示「No HLS URL found」或「no segment urls」？**
通常是该视频不可播放：已下架、仅会员/地区受限、或央视调整了该视频的接口。换一条视频验证；若整批都失败，可能是央视接口有调整，欢迎提 Issue。

**11. 部分片段解密失败 / 任务失败？**
多为网络波动。程序已对每个片段做最多 3 次指数退避重试；失败后**重试任务会自动从断点续传**，已完成的片段不会重新下载。持续失败请检查网络或代理。

**12. 下载很慢？**
在「设置 → 并发下载数」适当调高（默认 8，上限 16）。速度也受央视 CDN 与本地带宽限制。

**13. 合并很慢或卡顿？**
默认的「无损快速」很快。极少数片源在该模式下播放异常，可在「设置 → 合并方式」切到「兼容重编码」，速度变慢但兼容性最好。

**14. 海外访问下载失败？**
央视视频 CDN 可能对部分地区限流或不可达；这与本工具无关，可尝试更换网络环境。

## 📝 日志与反馈

**15. 日志在哪里？**
默认在应用的用户数据目录下（`cctvdl.log`、失败批次 `cctvdl-failures.log`）。可在「设置 → 日志目录」自定义位置，失败日志会写到同一目录。

**16. 如何反馈问题？**
请带上版本号（关于弹窗可见）、操作系统、复现步骤与相关日志，按 [Bug 模板](../.github/ISSUE_TEMPLATE/bug_report.yml) 提交 Issue。复现前可在「设置 → 日志级别」切到「调试」以记录更详细的日志；提交时注意脱敏。
